$t\bar{t}$-analysis¶

The chapter is covering the $t\bar{t}$-analysis logic. 2015 CMS Open Data is assumed as input data for concreteness (see Links for download section). The basic idea of this chapter is to give a basic algorithm for distinguishing t-quark pair production channel among other concurrent channels (production of single t-quark and w-jet) and to get the mass peak of t-quark using a histogram.

Input¶

Input data is a five sets of root-files. Each set is produced in MC simulation and represents partial interaction channel, one of five: $t\bar{t}$-channel, single top s-channel, single top t-channel, single top tW-channel, Wjets-channel. The root-file structure can be represented as a schematic: \ \ \ This diagram shows only those fields that will be required for further analysis such as electron, muon and jets. Each of this branches has its own hit number ($N_e$, $N_{\mu}$, $N_{jet}$), the transverse momentum value ($P _{\mathrm{T}}$} that will be used in the next sections for events filtering. Additionaly, jets have b-tag value that means whether or not a jet is b-tagget, i. e. produced by b-quark.

Output¶

The analysis task is a selection events from whole input data set, in which measured quantities were originated from $t\bar{t}$-decay. In real data one can not exatly know in which event $t\bar{t}$-pair were produced. But since we have simulated data, we know exactly whether or not $t\bar{t}$ in any particular event. That is defined by set to which file belongs, as was mentioned above. As five channels were involved, five different sets were generated and are given as input data (ttbar, single_top_s_chan, single_top_t_chan, single_top_tW, wjets). To select events we will apply some criteria (explained below) and then compare relative rate of those events which indeed were generated with $t\bar{t}$-quark pair production. The example below demonstrate successfully performed $t\bar{t}$ analysis. That is concludet from the fact, that most of the selected events indeed belong to $t\bar{t}$-channel.

Events filtering algorithm¶

Not all particles with it's quantities are needed for doing analysis. The only ones that is needed to applying some selection criterias are leptons (electrons and muons) and jets that is the products of $t\bar{t}$ decay. Since semi-leptonic decay is dominant process, two jets, two b-jets and one outgoing lepton are expected to be detected, as can be concluded from the diagram below:

The algorithm how to select only appropriative events is presentet below:

1. Filtering events by transverse momentum value¶

The transverse momentum ($P_\mathrm{T}$) of electrons, muons and jets must be higher then some threshold. The reason to select events with hight enought $P_\mathrm{T}$ is that the outgoing after hadron collision particles are almost always away from the beam-line in a significant amount. \ It is well suggestion to put threshold value 25 GeV. One can test any other value, but it must be hight enought to distinguish collision events from diffractive scattering events and must not be too large to avoid lossing significant amount of interesting events. \ At this step one have all simulated events as input data. The output is only those events which contains at least one electron, muon and jet with transverse momentum higher then 25 GeV.

2. Filtering events by lepton and jet numbers¶

Further, events that contain only one lepton (exactly one electron or exactly one muon) and four or more jets must be selected, as four jets and one lepton are produced after $t\bar{t}$-annihilation.

3. Filtering events by jet b-tagging¶

In the next, events that contain at least two b-tagget jets must be selected. It is well suggestion to put b-tag threshold equal to 0.5. So, the output is only those events which contains at least two jets with b-tag value higher than 0.5.\ The b-tag threashold also is not fixed as well as $P_\mathrm{T}$

$t$-quark mass plotting¶

For this moment one had to leave only appropriate events for our analysis. The final task is plotting $t$-quark mass. As following from $t\bar{t}$-decay diagram, $t$-quark decayed into b-quark, lepton and neutrino, while $\bar{t}$-quark into two quarks and one $b$-quark. $t$-quark mass can be restored by $\bar{t}$-quark decay products: one b-quark and two quarks, so three jets are needed for $t$-quark mass plotting.\ Consequantly, to perform mass plotting it is needed to find all possible trijet combinations per event and select the most appropriative one. To each event we have to assign trijet which is assumed to be the product of $\bar{t}$ decay.\ So to finish analysis it is needed:

  • Build all possible three-jet combinations per event.
  • Find the maximum b-tag value for each jet in each combination.
  • Leave only those combination with corresponding b-tag value more than selected threshold (> 0.5).
  • Select trijet with maximum transverse momentum. As output one has the sample of events with assigned trijet for each events.

At this point all is ready to plot trijet mass and to get a similar $t$-quark peak as showed above.

Summary¶

Here is brief cheat sheet for performing $t\bar{t}$ analysis:

1. Select events by following criterias:¶

  1. Transverse momentum is more then 25 GeV
  2. Single lepton, four or more jets
  3. At least two b-tagged jets (b-tag value threshold can be assigned to 0.5)

2. Build all possible trijet combinations per every event and leave single more appropriative by following criterias:¶

  1. At least one jet in three-jet set must have value more than threshold (0.5)
  2. Trijet $P_\mathrm{T}$ value mast be the largest among all combinations per event.
